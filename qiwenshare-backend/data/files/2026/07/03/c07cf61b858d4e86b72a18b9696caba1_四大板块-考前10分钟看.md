

## 一、基础云应用架构（35%）

### 1. 分层架构设计
- **典型三层架构**：前端接入层（SLB + ECS）、应用逻辑层（ECS/ACK）、数据存储层（RDS/OSS/Redis）。
- **高可用设计原则**：跨可用区部署、无单点故障、自动故障转移。

### 2. 核心产品配置与使用场景

#### （1）ECS（云服务器）
- **实例状态影响操作**：
  - **扩容系统盘**：无需重启，实例 ID 不变。
  - **更换系统盘**：必须停止实例，实例 ID 变更，公网 IP 不变（若为固定公网 IP），私网 IP 不变。
- **监控指标（ECS 控制台 Dashboard）**：
  - 基础监控：CPU 使用率、网络带宽、磁盘 I/O（需注意：**内存使用率属于操作系统级别，需安装 CloudMonitor 插件才能采集**）。
- **私网 IP 扩展**：
  - 可通过绑定辅助网卡或为主网卡添加辅助私网 IP 实现多 IP。
- **本地盘 vs 云盘**：
  - **本地 SSD 盘**：高性能、低延迟，但**不支持快照、不支持跨实例迁移、必须与 ECS 同一可用区**，生命周期随实例终止而销毁。
  - **ESSD 云盘**：支持 PL0/PL1/PL2/PL3 性能等级，性能与容量正相关，支持快照和跨可用区复制。

#### （2）OSS（对象存储）
- **上传限制**：
  - 简单上传 ≤ 5 GB；>5 GB 必须使用分片上传（支持断点续传）。
  - 单个 Object 最大 48.8 TB。
- **权限控制**：
  - 通过 RAM 授权或 Bucket Policy 控制访问。
  - 可生成**临时访问 URL（带签名+过期时间）**，实现临时授权。
- **回源配置**：
  - **镜像回源**：当 OSS 无文件时，自动从源站拉取并缓存。
  - **重定向回源**：返回 302 跳转到源站。
- **静态网站托管**：
  - 支持绑定自定义域名（CNAME），**无需备案即可托管静态页面**，但若用于中国大陆访问的域名，则域名本身仍需 ICP 备案。
  - 支持 HTTPS（可托管证书）。
- **图像处理（IMG）**：
  - 支持 resize、crop、rotate 等，但**resize 后不能放大原图**（会模糊），**不支持 OCR、人脸识别等 AI 功能**（属智能媒体服务范畴，非 OSS 基础功能）。

#### （3）RDS（云数据库）
- **存储类型**：
  - ESSD 云盘（推荐）、SSD 云盘、本地 SSD 盘。
  - **变更存储类型必须在“运行中”状态操作**。
- **读写分离**：
  - 主实例处理写请求，只读实例处理读请求，通过内网地址连接，**自动负载均衡**。
- **备份与恢复**：
  - 支持物理备份（快照）和逻辑备份（SQL dump）。

#### （4）CDN（内容分发网络）
- **组成模块**：
  - 链路调度系统、缓存节点、支撑系统（日志、监控、配置管理）。
- **加速域名要求**：
  - **中国大陆加速域名必须完成 ICP 备案**。
- **计费说明**：
  - 回源流量计费，但**OSS 作为源站在同地域回源走内网，不产生外网流量费**（仅收请求次数费）。

#### （5）智能接入网关（SAG）
- **作用**：实现**线下 IDC / 分支机构 / 移动办公**安全接入阿里云 VPC。
- **形态**：硬件设备（SAG-100WM）或软件客户端（SAG App）。
- **适用场景**：混合云组网，替代传统专线或 VPN。

## 二、复杂云网络架构（25%）

### 1. VPC 与经典网络对比
| 项目 | VPC | 经典网络 |
|------|-----|--------|
| 网络隔离 | 二层逻辑隔离 | 共享阿里云底层网络 |
| IP 规划 | 用户自定义（10.0.0.0/8 等） | 阿里云统一分配 |
| 安全控制 | 安全组 + 网络 ACL | 仅安全组 |
| EIP 支持 | ✅ | ❌（经典网络 ECS 使用固定公网 IP，不可解绑） |

### 2. VPC 核心组件
- **可用区（Zone）**：交换机（vSwitch）必须绑定到具体可用区。
- **保留 IP**：每个交换机的 `.0`、`.255` 以及最后三个 IP（如 /24 网段中 `.252`~`.255`）为系统保留。
- **路由表**：
  - 一个 VPC 可有多个自定义路由表。
  - **一个交换机只能绑定一个路由表**。
  - 下一跳类型包括：ECS、ENI、VPN 网关、NAT 网关、IPv6 网关等。

### 3. 混合云互联方案
- **IPsec-VPN**：
  - **网络对网络**打通（IDC ↔ VPC）。
  - **基于路由模式**（非策略模式），需本地网关有**静态公网 IP**。
  - 一个 VPN 网关可配置多个 IPsec 连接。
  - 可通过 **CEN（云企业网）** 实现多 VPC + 多 IDC 全互联。
- **SSL-VPN**：
  - **人对网络**接入，适用于移动办公。
  - 通过 SAG App 或 SSL-VPN 网关实现。

### 4. DNS 与解析
- **PrivateZone**：
  - 用于 VPC 内私有域名解析。
  - **每个地域默认有一个系统创建的 PrivateZone，不占用用户配额**。
- **公网解析**：需域名备案（中国大陆访问场景）。

## 三、云上高可用高性能架构（25%）

### 1. 负载均衡 SLB
- **类型对比**：
  | 类型 | 协议支持 | 适用场景 |
  |------|--------|--------|
  | ALB（应用型） | HTTP/HTTPS/gRPC | 七层 Web 应用，支持高级路由 |
  | NLB（网络型） | TCP/UDP/TLS | 四层高性能，百万级并发 |
  | CLB（传统型） | TCP/UDP/HTTP/HTTPS | 兼容旧版，逐步被 ALB/NLB 替代 |
- **健康检查**：
  - 检查源 IP 为 **100.64.0.0/10** 网段（阿里云保留网段）。
  - 默认检查间隔 2 秒，超时 5 秒。
- **高可用容灾**：
  - 多 SLB 实例可通过 **DNS 轮询**实现跨地域容灾。
  - **主备切换后不可自动回切**（避免脑裂）。

### 2. 弹性伸缩（Auto Scaling, AS）
- **伸缩组要求**：
  - 加入的 ECS 必须处于 **运行中（Running）** 状态。
  - 支持生命周期挂钩（Lifecycle Hook），用于自定义扩缩容前后的操作（如日志清理）。
- **触发方式**：定时任务、云监控指标（CPU > 70%）、API 手动触发。

### 3. 数据迁移与同步
- **DTS（数据传输服务）**：
  - 支持结构迁移、全量数据迁移、增量同步。
  - **支持 OLTP 和 OLAP 数据库**（如 RDS → AnalyticDB）。
  - 支持异构数据库（MySQL → Oracle 等）。

### 4. 容器与中间件
- **ACK（容器服务 Kubernetes 版）**：
  - 支持自动扩缩容（HPA）、服务发现、Ingress 流量管理。
- **Redis**：
  - 支持主从架构、集群版（分片），用于缓存加速。

### 5. 监控与告警
- **云监控（CloudMonitor）**：
  - **免费提供基础监控**（ECS、RDS、SLB 等）。
  - **支持电话报警**（SLB 自身监控不支持电话告警，但通过云监控可实现）。
  - 支持非阿里云服务器监控（需安装 Agent）。

## 四、云上安全体系架构（15%）

### 1. 安全组 vs 网络 ACL
| 特性 | 安全组 | 网络 ACL |
|------|--------|---------|
| 作用对象 | **ECS 实例的网卡** | **交换机（子网）** |
| 规则方向 | 有状态（允许返回流量） | 无状态（需显式放行双向） |
| 优先级 | 规则按顺序匹配 | 规则按数字优先级匹配（数字越小优先级越高） |

### 2. DDoS 防护
- **基础防护**：免费，自动启用（针对公网 IP）。
- **高防 DDoS（增强版）**：
  - **一旦开通，无法关闭或降级**。
  - 提供 Tbps 级防护能力。

### 3. 主机安全
- **云安全中心（原安骑士）**：
  - **免费版功能**：病毒木马查杀、异地登录告警、密码暴力破解检测、高危漏洞扫描。
  - **不包含 Web 应用层攻击防护**（如 WebShell 上传检测需付费版）。
- **安全管家**：
  - 提供人工安全巡检、漏洞修复建议（**自动扫描高危漏洞，无需管理员手动触发**）。

### 4. Web 应用防火墙（WAF）
- **防护能力**：
  - **注入类**：SQL 注入、命令注入。
  - **文件类**：WebShell 上传、本地/远程文件包含。
  - **脚本类**：XSS、CSRF。
- **白名单功能**：
  - Bot 管理白名单、数据安全白名单、CC 攻击豁免 IP。
- **部署模式**：透明接入（CNAME 接入）或独享集群。

### 5. 合规与应急
- **先知计划（安全众测）**：
  - 联合白帽子进行渗透测试。
  - 符合《网络安全法》要求，适用于等保三级及以上系统。
